# 美妆店智慧管理系统 - 模块详细设计文档

## 1. 模块概述

本文档详细描述美妆店智慧管理系统各核心模块的功能规格、业务流程、数据结构及接口定义。

---

## 2. 用户模块 (User Module)

### 2.1 功能概述

用户模块是系统的基础模块，负责用户的认证、授权和基本信息管理。系统支持三种角色：管理员(admin)、员工(staff)、顾客(customer)。

### 2.2 功能列表

| 功能 | 说明 |
|------|------|
| 用户登录 | 基于用户名、密码、角色的认证 |
| 用户注册 | 支持顾客自助注册 |
| 个人信息管理 | 查看和修改个人信息 |
| 密码修改 | 修改登录密码 |
| 角色切换 | 多角色用户切换 |

### 2.3 业务流程

```
用户登录流程：
┌─────────┐    ┌─────────────┐    ┌──────────────┐    ┌─────────┐
│ 输入账密 │───▶│ 验证用户   │───▶│ 生成Token   │───▶│ 返回结果 │
└─────────┘    └─────────────┘    └──────────────┘    └─────────┘
                     │                    │
                     ▼                    ▼
               ┌──────────┐         ┌──────────┐
               │ 验证失败 │         │  返回Token │
               │ 返回错误 │         │  跳转首页  │
               └──────────┘         └──────────┘
```

### 2.4 数据结构

**User 实体：**
```java
public class User {
    Long id;           // 用户ID
    String username;    // 用户名
    String password;    // 密码（BCrypt加密）
    String role;        // 角色：admin/staff/customer
    String name;        // 姓名
    String phone;      // 手机号
    String avatar;     // 头像URL
    LocalDateTime createdAt;  // 创建时间
    LocalDateTime updatedAt;  // 更新时间
}
```

### 2.5 业务规则

- 用户名唯一，长度4-20位
- 密码长度6-20位，使用BCrypt加密存储
- 手机号唯一，格式验证
- 不同角色看到不同的菜单和功能

## 3. 员工模块 (Employee Module)

### 3.1 功能概述

员工模块管理店铺员工信息，包括员工档案、职位管理、技能标签等。

### 3.2 功能列表

| 功能 | 说明 |
|------|------|
| 员工档案管理 | 增删改查员工基本信息 |
| 职位管理 | 设置员工职位（美容师、高级美容师等） |
| 技能标签 | 管理员工擅长的服务项目 |
| 状态管理 | 员工状态：在职/离职/休假 |
| 员工查询 | 按条件筛选查询员工 |

### 3.3 业务流程

```
员工入职流程：
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 创建用户 │──▶│ 完善档案 │──▶│ 设置职位 │──▶│ 分配技能 │
└──────────┘   └──────────┘   └──────────┘   └──────────┘
                                                            │
┌──────────┐   ┌──────────┐   ┌──────────┐                 │
│ 创建账号 │◀──│ 开通账号 │◀──│ 审核确认 │◀────────────────┘
└──────────┘   └──────────┘   └──────────┘
```

### 3.4 数据结构

**Employee 实体：**
```java
public class Employee {
    Long id;                    // 员工ID
    Long userId;                 // 关联用户ID
    String position;             // 职位
    LocalDate entryDate;         // 入职日期
    String employeeId;          // 员工工号
    String idCard;               // 身份证号
    String emergencyContact;     // 紧急联系人
    String emergencyPhone;       // 紧急联系电话
    String status;               // 状态：在职/离职/休假
    String skillTags;            // 擅长项目标签
}
```

### 3.5 业务规则

- 员工工号唯一，格式：EMP+数字
- 一个用户只能关联一个员工
- 员工离职后，用户状态不变，仅标记员工状态
- 技能标签以逗号分隔存储

```
新顾客接待流程：
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
```

## 5. 预约模块 (Reservation Module)

### 5.1 功能概述

预约模块管理服务预约，支持顾客自助预约和前台代约两种方式。

### 5.2 功能列表

| 功能 | 说明 |
|------|------|
| 创建预约 | 选择员工、时间、服务创建预约 |
| 预约确认 | 员工确认预约时间 |
| 预约取消 | 顾客或员工取消预约 |
| 预约状态管理 | 状态流转：待确认→已确认→已到店→已完成 |
| 预约提醒 | 预约时间前提醒顾客和员工 |
| 冲突检测 | 检测员工和时间段冲突 |

### 5.3 业务流程

```
预约服务流程：
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 选择服务 │──▶│ 选择美容师│──▶│ 选择时间 │──▶│ 确认预约 │
└──────────┘   └──────────┘   └──────────┘   └──────────┘
                                                      │
                                                      ▼
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│  服务完成 │◀──│  到店服务 │◀──│   到店   │◀──│  发送通知│
└──────────┘   └──────────┘   └──────────┘   └──────────┘
```

### 5.4 数据结构

**Reservation 实体：**
```java
public class Reservation {
    Long id;                    // 预约ID
    Long customerId;            // 顾客ID
    Long employeeId;            // 员工ID
    Long serviceId;             // 服务ID
    Long roomId;                // 房间ID
    LocalDateTime reservationDate; // 预约时间
    String status;              // 状态
    String source;              // 来源
}
```

### 5.5 状态流转

```
                    ┌──────────┐
                    │  待确认   │
                    └────┬─────┘
                         │
         ┌───────────────┼───────────────┐
         ▼               ▼               ▼
   ┌──────────┐   ┌──────────┐    ┌──────────┐
   │ 已确认   │   │ 已取消   │    │  爽约    │
   └────┬─────┘   └──────────┘    └──────────┘
        │
        ▼
  ┌──────────┐
  │  已到店  │
  └────┬─────┘
       │
       ▼
  ┌──────────┐
  │  已完成  │
  └──────────┘
```

### 5.6 业务规则

- 同一员工同一时间只能有一个预约
- 同一房间同一时间只能有一个预约
- 预约时间需在营业时间内
- 预约提前取消有时间限制
- 多次爽约顾客标记提醒

## 6. 订单模块 (Order Module)

### 6.1 功能概述

订单模块处理服务项目和商品的订单，支持多种支付方式和优惠抵扣。

### 6.2 功能列表

| 功能 | 说明 |
|------|------|
| 创建订单 | 选择服务/商品，创建订单 |
| 支付处理 | 支持多种支付方式 |
| 优惠抵扣 | 支持优惠券、积分抵扣 |
| 订单退款 | 支持部分退款和全额退款 |
| 订单查询 | 多条件查询订单 |

### 6.3 业务流程

```
订单处理流程：
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 选择项目 │──▶│ 应用优惠 │──▶│ 选择支付 │──▶│ 确认支付 │
└──────────┘   └──────────┘   └──────────┘   └──────────┘
                                                      │
                                                      ▼
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│  更新库存 │◀──│ 计算提成 │◀──│  保存订单│◀──│  收银结算│
└──────────┘   └──────────┘   └──────────┘   └──────────┘
```

### 6.4 数据结构

**Order 实体：**
```java
public class Order {
    Long id;                    // 订单ID
    String orderNo;              // 订单编号
    Long customerId;            // 顾客ID
    Long employeeId;            // 服务员工ID
    LocalDateTime orderDate;    // 订单时间
    BigDecimal totalAmount;     // 总金额
    String paymentMethod;       // 支付方式
    String status;              // 状态
}
```

**OrderItem 实体：**
```java
public class OrderItem {
    Long id;                    // 订单项ID
    Long orderId;               // 订单ID
    String itemType;            // 项目类型：service/product
    Long itemId;                // 项目ID
    Integer quantity;           // 数量
    BigDecimal price;           // 单价
}
```

### 6.5 支付方式

| 支付方式 | 说明 |
|----------|------|
| 现金 | 现金支付 |
| 微信 | 微信支付 |
| 支付宝 | 支付宝支付 |
| 银行卡 | 银行卡支付 |
| 储值余额 | 使用会员卡余额支付 |

### 6.6 业务规则

- 订单号格式：ORD+日期+序号
- 使用储值余额支付时检查余额是否充足
- 使用优惠券需检查优惠券有效性
- 积分抵扣：1积分=0.01元
- 退款时原路返回

## 9. 会员模块 (Membership Module)

### 9.1 功能概述

会员模块管理会员信息和会员卡，支持多种卡类型和会员权益。

### 9.2 功能列表

| 功能 | 说明 |
|------|------|
| 会员管理 | 会员信息管理 |
| 会员卡办理 | 办理各种类型会员卡 |
| 会员卡充值 | 储值卡充值 |
| 余额查询 | 查询会员卡余额和积分 |

### 9.3 卡类型说明

| 卡类型 | 说明 | 功能 |
|--------|------|------|
| 储值卡 | 预先充值一定金额 | 消费时扣除余额 |
| 疗程卡 | 包含特定服务次数 | 消费时扣除次数 |
| 折扣卡 | 享受指定折扣 | 消费时按折扣计算 |
| 时段卡 | 在特定时间段有效 | 在有效期内使用 |
| 组合卡 | 综合多种功能 | 结合储值和折扣 |

### 9.4 业务流程

```
会员卡办理流程：
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐

储值卡充值流程：
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 选择会员卡│──▶│ 输入金额 │──▶│ 确认支付 │──▶│ 更新余额 │
```

### 9.5 数据结构

```java
    Integer points;              // 积分余额
    LocalDateTime joinDate;     // 加入日期

```

**MembershipCard 实体：**
```java
public class MembershipCard {
    Long id;                    // 会员卡ID

    BigDecimal balance;          // 储值余额
    Integer remainingTimes;      // 剩余次数

    LocalDateTime startDate;    // 生效日期
    LocalDateTime endDate;       // 失效日期
    String status;               // 状态
}
```

### 9.6 会员等级

| 等级 | 折扣率 | 积分倍率 | 升级条件 |
|------|--------|----------|----------|
| 普通会员 | 100% | 1倍 | 注册即送 |
| 银卡会员 | 95% | 1.2倍 | 累计消费满1000 |
| 金卡会员 | 90% | 1.5倍 | 累计消费满5000 |
| 钻石卡会员 | 85% | 2倍 | 累计消费满20000 |

### 9.7 业务规则

- 会员卡号唯一，格式：MEM+日期+序号
- 储值卡余额不能为负
- 疗程卡次数不能为负
- 折扣卡折扣率范围：50%-100%
- 积分获取：每消费1元积1分（不同等级倍率不同）
- 积分兑换：100积分=1元

优惠券模块管理各种优惠活动，支持满减券、折扣券、代金券等。

### 10.2 功能列表

| 功能 | 说明 |

| 优惠券发放 | 向顾客发放优惠券 |
| 优惠券领取 | 顾客自助领取 |

| 类型 | 说明 | 示例 |

| 代金券 | 抵扣指定金额 | 50元代金券 |

### 10.4 数据结构

**Coupon 实体：**

### 10.5 业务规则

- 优惠券不找零、不兑换现金
